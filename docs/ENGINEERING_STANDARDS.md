# Engineering Standards

## Objective

Build maintainable, reusable and scalable software.

---

## Architecture

- Feature-first architecture.
- UI and business logic must remain separate.
- Reusable widgets belong in shared only when used by multiple features.
- Never duplicate workflows.

---

## Flutter

- Material 3 only.
- Use AppTheme.
- No hardcoded colors.
- No hardcoded text styles.
- Keep widgets small and reusable.

---

## Coding Rules

One responsibility per class.

One responsibility per file.

Prefer composition over duplication.

Keep functions short.

Meaningful names only.

---

## Teacher UX Rules

Teachers should complete most actions in one tap.

Use Default + Exceptions.

Avoid typing wherever possible.

Large touch targets.

Minimal scrolling.

---

## Parent UX Rules

Simple.

Warm.

Trustworthy.

No fictional content.

No unnecessary information.

Photos are the primary content.

---

## AI Rules

ChatGPT

- Product decisions
- Architecture
- Code review
- Sprint planning

Codex

- Small implementation tasks
- Maximum 1–2 files per task
- No architecture decisions

Claude

- Documentation
- Research
- Code review
- Backend design

---

## Definition of Done

Every completed task must:

- Compile successfully
- Be formatted
- Pass analyzer
- Follow existing architecture
- Include a short implementation summary

---

These standards apply to every future feature.
